# Quick Actions

## Purpose

Quick actions are the fastest way for a cook or sous-chef to record what happened during a shift.

They are not generic shortcuts. They are structured operational signals.

## Who Uses Them

### Cook

- product is empty;
- one unit is left;
- station is blocked;
- ticket is delayed;
- sous-chef is needed;
- open my station checklist.

### Sous-chef

- confirm stock signal;
- create urgent task;
- update stop-list;
- add handover note;
- call staff or supplier.

### Chef

- create/approve stop-list event;
- create process update;
- create briefing;
- review activity history.

## UX Rules

- One tap should create a useful signal.
- The user should never need to write a long report during service.
- Critical actions are red.
- Warnings/in-progress actions are amber.
- Completed/confirmation actions are green.
- Every action must write to activity history.
- If an action needs confirmation, it should create a pending signal instead of silently changing final truth.
